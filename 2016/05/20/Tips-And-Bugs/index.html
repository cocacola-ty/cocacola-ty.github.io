Bugs

2016-05-20

出现Duplicate Symbol链接错

  1. 可能该文件被引入两次，Targets的Build Phase设置项里，查看Complie Sources这一项，看看出现问题的类是不是有重复的，用文件名过滤也比较容易发现。这种问题一般也是多人协作开发时，project文件冲突导致的。发现这种问题，只要删除就可以了。不过有可能删除其中一个文件时，重复文件会自动全部删除，所以还需要单独添加一下。

  2. 出现问题的类是不是import .m文件

  3. 是不是第三方库中有文件和该类同名

webview进入didFailLoadWithError

error.code == -999

一个页面没有被完全加载之前收到下一个请求

可以直接return该错误尝试解决

translatesAutoresizingMaskIntoConstraints

  1. 缺少约束，约束冲突

  2. 少设置frame

出现访问僵尸对象问题

  1. 查看dealloc方法是否进行提前释放

  2. 查看dealloc方法调用时间

images.xcassests

  • 只支持png格式图片

  • images.xcassets中图片只能有 imagenamed:方法调取

通过在view上添加一个layer来实现动画，出现动画不还原

* 确保添加的是一个layer，不要重复一直添加layer，会覆盖。通过懒加载解决

多线程会导致的问题

  1. 开启子线程引起的内存开销，默认主线程1M，子线程512k

  2. 多线程之间由于网络请求时间差引发的问题

加载xib时报错

libc++abi.dylib:terminate_handler unexpectedly threw an exception

NSScanner: nil string argument

  1. 检查约束是否有冲突约束

  2. 检查是否有失效的连线！！ .m/.h 中已经删掉连线出来的控件，但是xib中未删除关联！！

下载图片时线程阻塞

dataWithContentsOfURL方法是同步方法，当图片较大或网络较差时 该方法阻塞后面其他代码会无法执行

调试时po不打印对象信息

将项目的scheme 改为Debug，release模式下不现实参数值
