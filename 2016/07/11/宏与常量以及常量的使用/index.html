宏与常量以及常量的使用

2016-07-11
#iOS

宏与常量的区别

  • 宏：在预处理器中进行文本替换，没有类型，不进行类型检查
  • 常量：共享一块内存空间，编译时会进行类型检查

更推荐使用常量

常量

const修饰位置不同的常量

1. const NSString *TYConst = @””;

const 修饰的是 TYConst，即 TYConst是不可以被修改的。 *TYConst = @“” 会编译报错，而将TYConst这个指针指向其他地址是不会报错的。

  1. 定义一个整形常量指针 p

  2. 定义一个整形变量a

  3. 因为const修饰的是p，所以p指针是可以改变的，p是不可以改变的

  4. 所以，将p指针指向变量a的地址不会报错，但是修改指针所指向位置的值会报错

2. NSString const *TYConst = @””;

const修饰的也是TYConst 这种写法和上面的一种写法是一样的,作用也是相同的

3. NSString * const TYConst = @””;

const修饰 TYConst指针，即该指针指向地址是不可以被修改的。

OC 中字符串为对象类型，所以不会像基本数据类型一样可以修改指针所指向地址的值，只会重新指向其他字符串地址。所以

一般定义一个字符串常量不想被修改应该用这种方式。
